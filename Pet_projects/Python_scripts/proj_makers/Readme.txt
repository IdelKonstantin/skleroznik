1) Распаковываем сниппеты в какую-нибудь папку, например в:

/home/$username/docs/snippets

2) Даем им права на исполнение:

sudo su
cd /home/$username/docs/snippets
chmod +x *.*

3) Устанавливаем переменную среды для запуска сниппетов:

echo 'export PATH="$HOME/docs/snippets:$PATH"' >> ~/.bashrc
source ~/.bashrc

4) Проверим, что переменная среды добавлена:

echo $PATH

5) Использование:

5.1 Для создания исполняемого файла:

	new_proj --name <имя проекта> --dir <директория для создания проекта>

	Пример:
	new_proj --name coolExec --dir . (создание проекта с одноименным бинарником в текущей директории)

5.2 Для создания статической библиотеки:

	new_proj --name <имя проекта> --dir <директория для создания проекта> --type s

	Пример:
	new_proj --name coolExec --dir . --type s (создание статической библиотеки в текущей директории)


5.3 Для создания разделяемой библиотеки:

	new_proj --name <имя проекта> --dir <директория для создания проекта> --type d

	Пример:
	new_proj --name coolExec --dir . --type d (создание разделяемой библиотеки в текущей директории)

5.4 Для создания заголовочного файла/файла исходного кода (только в текущем каталоге):

	new_code --name <название файла> --type h
	new_code --name <название файла> --type s
